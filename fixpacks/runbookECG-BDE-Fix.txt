## Runbook: Correção de Erro “Falha na abertura das tabelas” no ECGV6 (BDE)

### 1. Problema
O sistema ECGV6 não abre e apresenta a mensagem:
```
Falha na abertura das tabelas da base de dados!!!
Caminho: \\ELCUN3-ECG\hw\
Directory is controlled by other .NET file.
Directory: \\ELCUN3-ECG\HW.NETDIR\
File: \\ELCUN3-ECG\HW.PDOXUSRS.LCK
```
O erro indica que o BDE (Borland Database Engine) não consegue localizar o diretório de rede (NETDIR) onde são armazenados os arquivos de bloqueio.

### 2. Causa Raiz
- No servidor onde o banco de dados reside, existe um compartilhamento (ex.: `hw`) que aponta para `C:\HW\Database` e um subdiretório `NetDir` com o arquivo `PDOXUSRS.NET`.
- Nos clientes, a configuração `NETDIR` do BDE está ausente ou aponta para um caminho incorreto (ex.: `\\ELCUN3-ECG\HW.NETDIR\`).
- Sem essa entrada, o BDE tenta usar um local padrão que não existe, gerando o erro.

### 3. Diagnóstico Rápido
1. **Verifique a variável `HW_CAMINHO_DB`** no cliente:
   - Abra um PowerShell e digite: `$env:HW_CAMINHO_DB`
   - O valor esperado é o UNC do banco, por exemplo `\\ELCUN3-ECG\hw`.

2. **Teste o acesso ao diretório NetDir**:
   - No cliente, execute: `dir \\servidor\hw\NetDir` (substitua pelo nome do servidor e compartilhamento).
   - Se o diretório existir e conter `PDOXUSRS.NET`, o caminho está correto.

3. **Consulte o registro do BDE**:
   - Abra `regedit` e vá até `HKLM\SOFTWARE\WOW6432Node\Borland\Database Engine\Settings\SYSTEM\INIT`.
   - Verifique se existe o valor `NETDIR` e se ele aponta para o mesmo caminho testado acima.

### 4. Solução
**Execute os comandos abaixo em um PowerShell com privilégios de administrador** (no cliente afetado):

```powershell
# Caminho da chave do BDE
$regPath = "HKLM:\SOFTWARE\WOW6432Node\Borland\Database Engine\Settings\SYSTEM\INIT"

# Caminho correto do NETDIR (ajuste servidor/compartilhamento conforme seu ambiente)
$netDirCorreto = "\\ELCUN3-ECG\hw\NetDir"

# Cria ou atualiza a entrada
Set-ItemProperty -Path $regPath -Name "NETDIR" -Value $netDirCorreto -Type String -Force

# Confirma
Get-ItemProperty -Path $regPath -Name "NETDIR"
```

**Alternativa via CMD (também como admin):**
```cmd
reg add "HKLM\SOFTWARE\WOW6432Node\Borland\Database Engine\Settings\SYSTEM\INIT" /v NETDIR /t REG_SZ /d "\\ELCUN3-ECG\hw\NetDir" /f
```

**Após a alteração**, reinicie o ECGV6 (ou o computador) para que a nova configuração seja carregada.

### 5. Verificação
- Abra o ECGV6 novamente. Ele deve iniciar sem erros.
- Se o erro persistir, verifique se o arquivo `IDAPI32.CFG` (geralmente em `C:\Program Files (x86)\Common Files\Borland Shared\BDE\`) não contém uma definição antiga de `NET DIR`. Caso exista, edite o arquivo e corrija para o mesmo caminho UNC.

### 6. Observações Importantes
- A alteração é feita no **registro do computador** (`HKLM`), portanto vale para **todos os usuários** que acessarem aquela estação.
- O procedimento exige privilégios de administrador apenas no momento da configuração. Depois de feita, qualquer usuário comum conseguirá abrir o ECGV6.
- Se houver mais de um cliente com o problema, repita a correção em cada um.

### 7. Prevenção
- Mantenha um backup da chave do BDE antes de alterações:
  ```cmd
  reg export "HKLM\SOFTWARE\WOW6432Node\Borland\Database Engine\Settings\SYSTEM\INIT" C:\temp\bde_backup.reg
  ```
- Utilize o script automatizado `ECG-BDE-Fix.ps1` (fornecido separadamente) para diagnóstico e correção rápida.
