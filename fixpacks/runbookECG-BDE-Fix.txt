## Runbook: Correção de Erro “Falha na abertura das tabelas” no ECGV6 (BDE)

### 1. Problema
O sistema ECGV6 não abre e apresenta a mensagem:
```
Falha na abertura das tabelas da base de dados!!!
Caminho: \\ELCUN3-ECG\hw\
Directory is controlled by other .NET file.
Directory: \\ELCUN3-ECG\HW.NETDIR\
File: \\ELCUN3-ECG\HW.PDOXUSRS.LCK
```
O erro indica que o BDE (Borland Database Engine) não consegue localizar o diretório de rede (NETDIR) onde são armazenados os arquivos de bloqueio.

### 2. Causa Raiz
- No servidor onde o banco de dados reside, existe um compartilhamento que expõe a raiz lógica do banco.
- Dentro dessa raiz lógica existe o subdiretório `NetDir`, que deve conter o arquivo `PDOXUSRS.NET`.
- Nos clientes, a configuração `NETDIR` do BDE está ausente ou aponta para um caminho incorreto.
- Sem essa entrada, o BDE tenta usar um local padrão inválido e gera o erro.

### 3. Padrão operacional por unidade
O diretório do banco pode variar conforme a unidade. Para evitar hardcode espalhado, a suíte passa a usar o arquivo `ECG_UnitProfiles.json`.

Perfis padrão entregues:
- **UN1** → `\\SRVVM1-FS01\FS\ECG\HW\Database`
- **UN2** → `\\ELCUN2-ECG\hw`
- **UN3** → `\\ELCUN3-ECG\hw`

O `NETDIR` esperado é:
- **UN1** → `\\SRVVM1-FS01\FS\ECG\HW\Database\NetDir`
- **UN2** → `\\ELCUN2-ECG\hw\NetDir`
- **UN3** → `\\ELCUN3-ECG\hw\NetDir`

Se o caminho canônico de alguma unidade mudar, altere somente o arquivo `ECG_UnitProfiles.json`.

### 4. Execução sem comando manual de policy
Não é necessário rodar `Set-ExecutionPolicy` manualmente quando a execução for feita pelos wrappers:
- `ELCE_ECG_Diagnostics_Menu.bat`
- `ExecutarDiagnostico.cmd`
- `ECG-BDE-Fix_Menu.bat`

Esses wrappers já chamam `powershell.exe` com `-ExecutionPolicy Bypass`.

### 5. Diagnóstico rápido
1. Verifique a variável `HW_CAMINHO_DB` no cliente:
   - PowerShell: `$env:HW_CAMINHO_DB`
2. Verifique o registro do BDE:
   - `HKLM\SOFTWARE\WOW6432Node\Borland\Database Engine\Settings\SYSTEM\INIT`
3. Compare:
   - `NETDIR atual`
   - `NETDIR esperado pela unidade`
4. Valide se o `NetDir` existe e se contém `PDOXUSRS.NET`

### 6. Fluxo recomendado
1. Executar o laudo principal (`ELCE_ECG_Diagnostics`)
2. Se houver indício de problema BDE, abrir `ECG-BDE-Fix_Menu.bat`
3. Rodar primeiro em modo **diagnóstico**
4. Confirmar `NETDIR esperado`
5. Aplicar a correção
6. Reabrir o ECGV6
7. Se necessário, rerodar o laudo principal

### 7. Correção direta via script
O fix pack oficial é:
```powershell
.\ECG-BDE-Fix.ps1 -Fix
```

Se também quiser criar os diretórios locais padrão:
```powershell
.\ECG-BDE-Fix.ps1 -Fix -CreateMissingDirs
```

### 8. Observações importantes
- A alteração do `NETDIR` é feita em `HKLM`, portanto vale para todos os usuários da estação.
- O fix pack cria backup da chave do BDE antes de alterar o valor.
- A criação automática de `NetDir` remoto foi bloqueada por segurança. Se o caminho remoto não existir, o fix pack apenas sinaliza a anomalia.
- O core do laudo continua **read-only**.

### 9. Prevenção
- Manter o `ECG_UnitProfiles.json` atualizado por unidade
- Usar sempre os wrappers BAT/CMD
- Rerodar o laudo após qualquer correção
- Evitar alteração manual no BDE sem backup da chave
