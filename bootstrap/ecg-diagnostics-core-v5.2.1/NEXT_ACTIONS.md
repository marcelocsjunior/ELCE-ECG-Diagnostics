# Próximas ações recomendadas

1. Criar o novo repositório com o nome final escolhido.
2. Subir apenas a linha standalone do produto.
3. Copiar os 5 artefatos canônicos da release para a árvore final do novo repo.
4. Alinhar version strings internas do pacote para eliminar drift textual residual.
5. Regenerar `SHA256SUMS.txt` final.
6. Publicar a primeira tag `v5.2.1`.
7. Fazer validação controlada em Windows real.
8. Só então considerar rollout operacional mais amplo.
